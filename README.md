Інформаційний чат-бот для компанії ViknaDOMI, побудований на базі **aiogram 3.x**. Бот слугує цифровим каталогом з обмеженим доступом для партнерів та співробітників.

## 🚀 Основні можливості

### 🛡 Безпека та Доступ
- **Верифікація за номером:** Для доступу до функцій бота користувач має поділитися своїм контактом.
- **Модерація заявок:** Адміністратори отримують сповіщення про нові реєстрації та можуть схвалити або відхилити запит прямо в чаті.
- **Приватність:** Всі функції (меню, каталоги) заблоковані до моменту підтвердження адміном.
- **Командне меню:** Видимість команд (`/admin`) динамічно змінюється залежно від прав доступу.

### ⚙️ Адмін-панель
- **Керування контентом:** Додавання та видалення фото, відео, документі (PDF) безпосередньо через бот.
- **Broadcasting (Розсилка):** Надсилання повідомлень з медіафайлами всім зареєстрованим користувачам з отриманням звіту.
- **Керування користувачами:** Перегляд повного списку учасників, можливість миттєвого блокування/розблокування.
- **Швидка відповідь:** Адміни можуть відповідати на звернення користувачів, просто пересилаючи повідомлення (Reply).

### 📂 Каталог
- **Деревоподібне меню:** Зручна навігація за категоріями.
- **Мультимедіа галереї:** Динамічне гортання фото/відео з кнопками навігації.
- **Технічні файли:** Прямий доступ до PDF-каталогів та креслень.

## 🔧 Встановлення та запуск

1. **Клонування репозиторію:**
   ```bash
   git clone https://github.com/kamedashe/viknadomi.git
   cd VikNaDomi
   ```

2. **Налаштування оточення:**
   Створіть файл `.env` у корені проекту:
   ```env
   BOT_TOKEN=ваш_токен_від_BotFather
   ADMIN_IDS=id1,id2
   ```

3. **Встановлення залежностей:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Запуск бота:**
   ```bash
   python bot.py
   ```

## 📂 Структура проекту
- `bot.py` — точка входу, ініціалізація бота та роутерів.
- `database/` — моделі даних та асинхронні запити до БД.
- `handlers/` — логіка обробки повідомлень (admin, user, menu).
- `keyboards.py` — конструктори всіх кнопок та навігації.
- `menu_structure.py` — конфігурація дерева категорій каталогу.
- `states.py` — опис станів FSM (скінченних автоматів).
